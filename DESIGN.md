# 📋 AutoConfig Tool — 智能配置表格系统

## 一、项目概述

### 1.1 产品定位
企业级智能配置表格系统，面向系统集成商/设备供应商。根据客户需求（招标参数），结合厂家产品知识库，自动生成设备配置清单。对标 WPS 表格的交互体验，内嵌 AI 智能体实现配置自动化。

### 1.2 核心价值
- **人工配单 vs AI 配单**：传统人工翻阅产品手册配单需要数小时，AI 配单秒级完成
- **知识沉淀**：每次配单都是对 AI 的训练，越用越精准
- **分布式学习**：团队配单经验可共享，新人也能配出老手的水平

## 二、表格模板规范

### 2.1 列定义（18列 A-R）

| 列 | 列名 | 宽度 | 说明 |
|:--|:----|:----|:----|
| A | 序号 | 11字符 | 自动编号 |
| B | 设备名称 | 11字符 | 客户/招标需求中的设备名称 |
| C | 招标/需求参数 | 11字符 | 客户写明的技术参数要求 |
| D | 单位 | 11字符 | 需求单位（台/套/个） |
| E | 数量 | 11字符 | 需求数量 |
| F | *(空)* | 11字符 | 备用列 |
| G | *(空)* | 11字符 | 备用列 |
| H | 产品名称 | 11字符 | 厂家推荐产品名称 |
| I | 产品规格 | 11字符 | 厂家产品详细规格 |
| J | 品牌 | 11字符 | 品牌名 |
| K | 产品型号 | 11字符 | 厂家产品型号 |
| L | 厂家内部型号 | 11字符 | 内部编码/物料号 |
| M | 数量 | 11字符 | 配置数量 |
| N | 单位 | 11字符 | 配置单位 |
| O | 产品单价 | 11字符 | 单价（元） |
| P | 总价 | 11字符 | 总价 = M × O |
| Q | 备注 | 11字符 | 补充说明 |
| R | 不满足参数 | 11字符 | 需求参数 vs 产品规格的差异项 |

### 2.2 格式规范
- **列宽**：11个字符（约82px）
- **行高**：20磅（约27px）
- **字体**：宋体（SimSun）
- **字号**：10pt
- **对齐**：默认左对齐，数字列右对齐
- **边框**：细线灰色网格
- **颜色**：A-E列浅蓝底色（需求区），H-Q列浅绿底色（配置区），R列浅红底色（不满足项）

### 2.3 分区说明

```
┌─────────────────────┬─────────────────────────────────┬────────────┐
│     需求区 (A-E)     │         配置区 (H-Q)             │ 差异区 (R) │
│  客户招标原始内容    │  厂家对每个需求项的配置响应       │ 不满足项   │
│  灰色底色            │  绿色底色                       │ 红色底色   │
└─────────────────────┴─────────────────────────────────┴────────────┘
```

## 三、三大工作模式

### 模式 1: 学习模式 🎓

#### 3.1.1 目标
让 AI 理解厂家的产品体系、参数规格、定价策略，建立知识库。

#### 3.1.2 输入方式
- **上传产品配置模板**：Excel/CSV 格式的产品选型表
- **上传产品 FAQ**：常见问题文档（PDF/Word/Excel/TXT）
- **上传产品目录**：产品型号、规格、价格清单
- **手动导入**：直接在表格中粘贴产品数据
- **URL 导入**：从公司网站/网盘导入产品资料

#### 3.1.3 学习输出
- **知识库文件**：`knowledge_[产品线名称]_[版本].xlsx` — 结构化的产品参数表
- **记忆文件**：`memory_[用户ID].json` — AI 的学习经验和匹配策略
- **产品关系图谱**：产品之间的替代关系、配套关系

#### 3.1.4 自动提取字段
从产品资料中自动提取：
- 产品名称、品牌、型号、内部编码
- 技术参数（按维度分类：性能、物理、电气、环境等）
- 价格信息（单价、折扣等）
- 适用场景、行业认证
- 兼容性信息

### 模式 2: 半自动模式 🤖

#### 3.2.1 目标
AI 辅助人工配单，用户手动填入配置内容，AI 实时学习用户的配置逻辑。

#### 3.2.2 工作流程
1. 导入/粘贴需求内容到 A-E 列
2. 用户在 H-Q 列开始填写配置
3. AI 实时观察用户行为，学习用户如何将需求映射到产品
4. AI 在侧边 AI 对话框给出建议
5. 用户可接受/拒绝/修改 AI 建议
6. 用户可对 AI 进行「调教」— 纠正 AI 的错误匹配

#### 3.2.3 AI 对话框功能
- **可拖动**：按住标题栏拖到任意位置
- **可缩放**：拖动右下角调整大小
- **透明度可调**：滑块控制透明度（10%-100%）
- **可锁定**：锁定位置和大小，避免误操作
- **浮动在表格之上**：不遮挡表格主要内容
- **响应式内容**：
  - 知识库查询结果
  - 实时匹配建议
  - 用户操作历史
  - 产品详情卡片
  - 调教面板

### 模式 3: 全自动模式 🚀

#### 3.3.1 目标
一键完成全部配置，AI 独立完成从需求分析到配置生成的完整流程。

#### 3.3.2 工作流程
1. 导入需求内容
2. 点击「全自动配置」
3. AI 逐行分析需求 → 匹配产品 → 填写配置 → 标注不满足项
4. 进度条显示处理进度
5. 完成后高亮显示差异项（R列）

#### 3.3.3 智能匹配算法
- 需求相似度匹配（语义分析）
- 参数规格逐项对比
- 价格优化建议
- 替代产品推荐
- 配置完整性检查

## 四、AI 智能体架构

### 4.1 核心组件

```
┌─────────────────────────────────────────────┐
│              AI Agent System                  │
├─────────────────────────────────────────────┤
│  ┌─────────┐ ┌──────────┐ ┌────────────┐   │
│  │ 推理引擎 │ │ 学习引擎 │ │ 知识库管理器 │   │
│  └─────────┘ └──────────┘ └────────────┘   │
│  ┌─────────┐ ┌──────────┐ ┌────────────┐   │
│  │ 匹配引擎 │ │ 记忆管理器│ │ 同步管理器  │   │
│  └─────────┘ └──────────┘ └────────────┘   │
└─────────────────────────────────────────────┘
```

### 4.2 推理引擎
- 基于 LLM（支持 OpenAI/Claude/本地模型）
- 多轮对话上下文管理
- 工具调用（查询知识库、匹配产品、计算价格）

### 4.3 学习引擎
- 用户行为序列学习
- 需求-产品映射归纳
- 参数对标规则提取
- 反馈强化（用户纠正 = 正样本）

### 4.4 知识库管理器
- 层次化产品知识图谱
- 全文检索 + 向量检索（RAG）
- 参数标准化和维度对齐
- 版本管理和增量更新

### 4.5 记忆系统

#### 记忆文件结构

```
memory_[user_id].json
├── meta (版本、创建时间、用户信息)
├── matching_rules (匹配规则列表)
│   ├── input_pattern (需求关键词模式)
│   ├── product_match (匹配的产品)
│   ├── confidence (置信度)
│   └── source (来源: 学习/调教/自动)
├── parameter_mappings (参数映射)
│   ├── demand_param (需求参数名)
│   ├── spec_param (产品参数名)
│   ├── transform (转换规则)
│   └── examples (示例)
├── user_preferences (用户偏好)
│   ├── preferred_brands
│   ├── pricing_strategy
│   └── fallback_rules
└── sync_info
    ├── last_sync
    └── version
```

#### 记忆文件导出
- 用户可导出为 `.mem` 文件
- 其他电脑可导入 `.mem` 文件
- 记忆依赖的知识库文件单独导出为 `.kdb` 文件

### 4.6 分布式同步（可选）
- 云中心存储记忆
- 基于冲突解决策略（最后写入 + 置信度权重）
- 用户注册后开启同步
- 隐私保护：仅同步学习到的规则，不包含客户数据
- 可勾选「启用云同步」开关

## 七、技术架构

### 7.1 技术栈

| 层 | 技术 | 说明 |
|:--|:----|:----|
| **前端** | React 18 + Vite | 快速 HMR，现代化 SPA |
| **表格引擎** | 自定义 React 组件 | 内容可编辑 HTML 表格，模拟 Excel 行为 |

| **文件解析** | Pandas + python-docx + PyPDF2 | 多格式文件导入 |
| **同步** | WebSocket + REST API | 分布式记忆同步 |

## 八、扩展功能

### 8.1 产品深度学习
- 从产品手册自动提取参数维度
- 学习产品之间的替代关系（A可以替代B当参数X≥Y）
- 学习行业标准术语（不同厂家对同一参数的命名差异）

### 8.2 智能对比引擎
- 逐行对比需求参数 vs 产品参数
- 标出不满足的参数项（R列）
- 给出替代方案建议

### 8.3 自我进化（元编程）
- AI 可以新增功能模块
- AI 可以修改表格模板配置
- AI 可以添加新的分析维度

### 8.4 批量处理
- 一次导入多个需求表
- 批量生成配置清单
- 导出为带格式的 Excel 文件

### 8.5 报告生成
- 自动生成投标文件的技术偏离表
- 生成配置清单摘要报告
- 价格分析和优化建议

## 九、非功能性需求

### 9.1 性能
- 1000行以内的表格操作流畅无卡顿
- AI 匹配单条需求 < 3 秒
- 全自动配置 100 行 < 60 秒
- 记忆文件 < 10MB

### 9.2 安全
- 密码 bcrypt 加密
- 本地数据不自动上传
- 记忆同步可关闭
- 客户数据本地存储，不上传云端

### 9.3 兼容性
- Windows 7/10/11
- 支持 WPS Office / Microsoft Office 文件格式
- 支持 CSV、TXT 导入
